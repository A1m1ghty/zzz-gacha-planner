# 新艾利都资源规划局

面向《绝区零》限定代理人与音擎的非官方抽卡概率与资源规划器。输入现有菲林、加密母带、保底状态和目标组，即可估算全部目标达成率、分阶段成功率、典型所需抽数以及仍需补充的资源。

所有模拟都在浏览器本地完成，无需登录，不会上传账号或规划数据。

## 在线使用

**GitHub Pages：<https://a1m1ghty.github.io/zzz-gacha-planner/>**

推荐使用最新版 Chrome、Edge、Safari 或 Firefox。首次打开后即可直接填写数据；输入内容会自动保存在当前浏览器中。

## 能解决什么问题

- 我现有的菲林和母带一共能抽多少次？
- 以当前垫抽和大小保底状态，完成全部目标的概率是多少？
- 如果想达到 80%、90% 或 95% 的把握，还需要准备多少抽？
- 多个代理人和音擎目标按指定顺序抽取时，保底如何继承？
- 计入重复角色、音擎产生的信号余波后，资源缺口会减少多少？
- 当前资源不足时，所需抽数主要分布在哪些区间？

## 主要功能

### 资源与保底

- 录入菲林、单色菲林、加密母带和信号余波。
- 按 `160 菲林 = 1 抽`、`20 信号余波 = 1 张加密母带` 换算。
- 分别设置代理人池和音擎池的 S 级垫抽、下一个 S 级是否大保底，以及 A 级垫抽。
- 代理人池与音擎池各自继承垫抽和大保底，互不影响。

### 目标组

- 每个目标组可同时设置限定代理人和限定音擎目标。
- 支持从当前持有数量规划到目标影画或音擎星级。
- 支持普通限定池和“首次 S 级必得”的重映池。
- 可登记目标池中的两个 UP A 级代理人及其持有状态，用于估算重复返还。
- 目标组按页面顺序依次消耗资源，同类卡池状态会继续继承。

### 结果展示

- 全部目标达成率与成功样本数。
- 各目标的累计完成概率。
- 成功样本中位抽数、全部样本中位抽数和中位补抽。
- 平均、80%、90%、95% 概率所需抽数与资源缺口。
- 不依赖随机模拟的最坏硬保底抽数。
- 计入与不计入信号余波返还的对照。
- 95% 采样波动提示。
- 本次模拟使用的资源、目标与保底前提快照。
- 按抽数区间展示的达成/缺口分布、区间占比、累计占比和样本频数。

### 本机数据

- 自动保存到浏览器 `localStorage`。
- 支持 JSON 导入和导出，当前数据结构版本为 `schemaVersion: 1`。
- 导入非法或不支持的数据时会显示错误，不会覆盖现有规划。
- 提供重置按钮，可恢复初始示例数据。

## 使用方法

1. 在“现有资源”中填写菲林、母带和信号余波。
2. 在“目标组”中设置希望获得的代理人影画与音擎星级。
3. 在“保底状态”中填写两个限定池当前的 S/A 级垫抽及 S 级大保底状态。
4. 如需更精细的返还估算，在目标组高级设置和账号持有画像中补充数据。
5. 页面会自动运行 20,000 次快速预览；需要更稳定的结果时，可运行 100,000 次精确模拟。
6. 综合查看达成率、中位补抽、90%需求、硬保底和完整分布，不要只依赖单一数字。

## 结果应该如何理解

| 指标 | 含义 |
| --- | --- |
| 全部目标达成率 | 当前资源按目标顺序完成整个计划的模拟概率 |
| 逐目标累计达成率 | 完成当前目标以及它之前所有目标的概率 |
| 成功样本中位抽数 | 只统计现有资源内成功路径时的典型投入 |
| 所有样本中位抽数 | 所有模拟路径所需抽数的第 50 百分位 |
| 80% / 90% / 95% 所需 | 在模拟中覆盖对应比例路径所需准备的抽数 |
| 平均所需 | 所有模拟路径所需抽数的算术平均值，不等于 50% 成功率 |
| 最坏硬保底 | 按硬保底和大小保底确定性计算的理论上限，不计随机返还 |
| 达成与缺口分布 | 各抽数区间出现的频率；绿色表示当前资源可覆盖，黄色表示仍需补抽 |

蒙特卡洛结果存在抽样误差。固定种子让同一组输入能够得到可复现结果，但不代表实际抽卡必然按照该分布发生。

## 规则与计算模型

当前规则标记：**3.0 / 2026-07-12**。

### S 级限定目标

| 卡池 | 基础 S 级概率 | 硬保底 | 当期目标命中率 | 软保底近似 |
| --- | ---: | ---: | ---: | --- |
| 限定代理人 | 0.6% | 90 抽 | 50%；失败后下一个 S 级必为目标 | 第 74 抽起，每抽增加 6 个百分点 |
| 限定音擎 | 1.0% | 80 抽 | 75%；失败后下一个 S 级必为目标 | 第 65 抽起，每抽增加 7 个百分点 |

普通限定池会继承对应卡池的垫抽和大小保底。重映池的“首次 S 级必得”作为目标槽位的独立特殊保底处理，消耗后不会错误清除普通大保底状态。

> 逐抽软保底递增曲线并非官方公开数据，而是用于规划的近似模型。官方确认规则与近似参数已在网站中分开标注。

### A 级模型

- 代理人池 A 级基础概率采用 9.4%，最多 10 抽出现 A 级或以上，UP 命中率采用 50%。
- 音擎池 A 级基础概率采用 15%，最多 10 抽出现 A 级或以上，UP 命中率采用 75%。
- S 级结果优先，并视为满足“A 级或以上”保底。
- 非 UP A 级按代理人与音擎各 50% 处理，以匹配公开的细分类基础概率。

### 信号余波返还

| 抽取结果 | 模拟返还 |
| --- | ---: |
| A 级代理人，重复至 M6 期间 | 8 信号余波 |
| A 级代理人，满影画后溢出 | 20 信号余波 |
| S 级代理人，重复至 M6 期间 | 40 信号余波 |
| S 级代理人，满影画后溢出 | 100 信号余波 |
| A 级音擎 | 8 信号余波 |
| S 级音擎 | 40 信号余波 |

模拟路径中每累计 20 信号余波会立即兑换 1 张加密母带，并继续用于后续目标。当前版本不计算 B 级信号残响、月度限购商店或其他活动兑换。

### 模拟方式

- 使用固定种子的蒙特卡洛模拟，确保同一输入可以复现。
- 默认自动运行约 20,000 次快速预览，手动精确模拟运行约 100,000 次。
- 计算在浏览器 Web Worker 中进行，运行期间页面仍可操作，并可主动停止。
- 每条路径记录完成计划所需的最大外部投入，用于生成平均值、中位数、80/90/95 分位数和区间分布。
- 硬保底通过独立的确定性算法计算，不依赖随机抽样。

## 当前范围与限制

首版仅规划限定代理人与限定音擎，不包含：

- 常驻池、新人折扣和 300 抽自选
- 邦布池
- 当前卡池名单自动同步
- 活动收入、每日资源增长和版本资源预测
- 付费价格、充值档位或特殊邀约函兑换
- B 级信号残响和月度商店限购

UP A 级名称由用户自行填写，网站不依赖内置的当期卡池名单，因此不会因版本更新立即失效。

## 本地开发

### 环境要求

- Node.js 22.13 或更高版本
- npm

### 安装与启动

```bash
git clone https://github.com/A1m1ghty/zzz-gacha-planner.git
cd zzz-gacha-planner
npm install
npm run dev
```

本地开发服务默认可通过终端显示的地址访问。

### 常用命令

| 命令 | 用途 |
| --- | --- |
| `npm run dev` | 启动本地开发服务器 |
| `npm run build` | 生成 Sites/Vinext 生产构建 |
| `npm run build:pages` | 生成 GitHub Pages 静态文件到 `docs/` |
| `npm test` | 执行生产构建、模拟器测试和服务端渲染测试 |
| `npm run lint` | 运行 ESLint |

### 主要目录

```text
app/
├── Planner.tsx          # 页面交互与结果展示
├── globals.css          # 响应式视觉样式
├── sim.worker.ts        # 浏览器后台模拟入口
└── lib/
    ├── rules.ts         # 集中的概率与保底参数
    ├── simulator.ts     # 蒙特卡洛和硬保底计算
    └── types.ts         # TypeScript 数据结构
docs/                    # GitHub Pages 静态产物
tests/                   # 模拟器与渲染测试
```

## 构建与发布

GitHub Pages 使用 `docs/` 目录中的静态产物。修改应用代码后运行：

```bash
npm run build:pages
```

随后将源码和更新后的 `docs/` 一起提交。仓库的 Pages 来源应设置为 `main` 分支的 `/docs` 目录。
